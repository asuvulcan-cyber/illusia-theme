01 — Fundamento

A Filosofia do Design Illusia

O Illusia é um sistema visual construído em torno de uma ideia central: um livro raro de colecionador dentro de um observatório astronômico. Escuridão profunda com detalhes que brilham. Densidade controlada. Cada elemento justifica sua presença ou não existe.

Nome do estilo
Dark Editorial Observatory — a intersecção entre o refinamento tipográfico de uma publicação literária de luxo e a precisão silenciosa de um painel de controle científico. Nunca genérico. Nunca barulhento.
Os quatro pilares
✦ Atmosfera
Cada tela deve ter profundidade visual mesmo sem imagens. Ruído de grão, orbs de luz difusa, linhas de grade fantasmas e bordas hairline criam um ambiente vivo.
✦ Hierarquia
O olhar tem um caminho. Título serifado dramático → subtítulo mono sutil → corpo sans-serif leve. Nunca dois elementos disputando protagonismo no mesmo nível.
✦ Contenção
Menos é mais, mas menos errado é mais errado ainda. Cada cor, borda e sombra tem função semântica. Decoração sem propósito é ruído — não elegância.
✦ Fluidez
Nenhum valor de tamanho é fixo. Todo espaçamento, fonte e raio usa clamp() para se adaptar naturalmente de 375px a 1440px sem breakpoints artificiais.
✗ O que este design nunca é
Gradientes roxos genéricos. Fundos brancos. Fontes Inter/Roboto. Cards com sombras coloridas excessivas. Animações desnecessárias em elementos estáticos. Bordas coloridas fortes sem motivo semântico. Mais de 3 tons de acento em uso simultâneo.
02 — Fundamento

Tokens CSS

Todos os valores do sistema vivem como custom properties no :root. Nunca use valores literais no código — sempre referencie um token. Isso garante consistência e permite mudar o tema inteiro com poucos ajustes.

Estrutura do token
Os tokens seguem nomenclatura semântica em três camadas: categoria, escala e opcionalmente variante.
/* ✓ CORRETO — referencia token */ padding: var(--space-md); font-size: var(--text-sm); border-color: var(--border-amber); /* ✗ ERRADO — valor literal hardcoded */ padding: 12px; font-size: .875rem; border-color: rgba(232,184,109,.22);
Anatomia do clamp()
Todo valor fluido segue a fórmula: clamp(mínimo, preferido-em-vw, máximo). O valor central em vw é o que faz a escala ser contínua entre os extremos.
/* Fórmula padrão */ --space-md: clamp(.875rem, 2vw, 1.25rem); /* ↑ 375px ↑ cresce ↑ 1440px */ /* Raio de borda também é fluido */ --r-xl: clamp(12px, 1.8vw, 20px);

✗ Nunca use cores de acento livremente
Teal = status "em andamento" e gêneros. Crimson = avisos, conteúdo adulto, erros. Violet = fandoms. Sage = "concluído". Âmbar = todo o resto. Não inverta esses papéis por estética.
Regra dos 3 tons
✓ Regra obrigatória
Nunca mais de 3 tons de acento em uso visível simultâneo numa mesma tela. Em geral: âmbar + teal + neutros. Crimson e violet aparecem apenas quando há semântica para isso (aviso, fandom).

06 — Fundamento

Raio de Borda

Cinco níveis fluidos que definem a "personalidade" de cada elemento. O raio comunica hierarquia: elementos menores e utilitários têm raio menor; containers maiores e mais importantes têm raio maior. Nunca use o mesmo raio para elementos de hierarquias diferentes.

Escala completa
--r-xs clamp(2px, .3vw, 3px)
Badges, pills, chips, new-badge inline
--r-sm clamp(3px, .5vw, 5px)
Botões icon, ctrl-btn, status badges, sort-btn
--r-md clamp(6px, .9vw, 9px)
Botões de ação, inputs, capa do cover, chapter-list
--r-lg clamp(9px, 1.4vw, 14px)
Char cards, listas de capítulos, demo-frame
--r-xl clamp(12px, 1.8vw, 20px)
Cards principais, story-card, modais, sidebar-card
✓ Raio e hierarquia andam juntos
Quanto maior o container na hierarquia visual, maior o raio. Um badge (--r-xs) dentro de um card (--r-xl) cria contraste intencional que reforça o aninhamento.
✗ Nunca arredondamento total
Pills e botões nunca usam border-radius: 9999px (pílula). O sistema é editorial, não playful. O máximo permitido é --r-xl, exceto em avatares circulares.
✓ Exceção: avatares
Avatares de personagem e de usuário usam border-radius: 50%. São os únicos elementos circulares do sistema — e essa exclusividade torna o círculo semanticamente significativo: representa uma pessoa.
06 — Atmosfera

Texturas & Ruído

O grão de ruído é aplicado globalmente via body::before com position: fixed. Ele cria a sensação de materialidade física e quebra a artificialidade do digital.

/* Grain global — sempre em position: fixed para cobrir a viewport */ body::before { content: ''; position: fixed; inset: 0; background-image: url("data:image/svg+xml, ...feTurbulence..."); pointer-events: none; z-index: 9999; /* acima de tudo */ opacity: .45; /* sutil — deve ser quase imperceptível */ }
✓ Calibração do ruído
baseFrequency entre .8 e .9 para grão fino. numOctaves: 4 para textura rica. Opacidade global entre .35 e .55 — deve ser sentido, não visto conscientemente.
07 — Atmosfera

Orbs de Luz Ambiente

Gradientes radiais difusos animados em position: fixed criam profundidade e calor sem poluir o conteúdo. Invisíveis conscientemente, presentes atmosfericamente.

/* Estrutura dos orbs */ .orb { position: fixed; border-radius: 50%; filter: blur(clamp(60px, 10vw, 120px)); /* blur fluido! */ pointer-events: none; z-index: 0; animation: orb-drift 25s ease-in-out infinite; } /* Orb âmbar — sempre presente, top-left */ .orb-amber { background: radial-gradient(circle, rgba(232,184,109,.07) 0%, transparent 70%); } /* Nunca mais de 3 orbs. Opacidade máxima: .08 */ @keyframes orb-drift { 0%, 100% { transform: translate(0,0) scale(1); } 33% { transform: translate(3%, 5%) scale(1.05); } 66% { transform: translate(-3%,-3%) scale(.96); } }

/* Receita completa do card */ .card { background: var(--glass-bg); /* rgba(22,18,25,.7) */ backdrop-filter: var(--glass-blur); /* blur(clamp(8px,1.5vw,16px)) */ border: 1px solid var(--glass-border); /* rgba(255,240,210,.08) */ border-radius: var(--r-xl); overflow: hidden; position: relative; /* âncora o ::before */ transition: border-color var(--t-mid), transform var(--t-mid) var(--ease-expo); } .card:hover { border-color: var(--border-amber); transform: translateY(-1px); } /* Shimmer line — obrigatório */ .card::before { content: ''; position: absolute; top: 0; left: 10%; right: 10%; height: 1px; background: linear-gradient(90deg, transparent, rgba(232,184,109,.12), transparent); pointer-events: none; }
Illusia-card — grid de listagem
O card de obra usa CSS Grid de 2 colunas: cover fixo (col 1, spana todas as rows) + conteúdo (col 2). É o componente mais complexo do sistema.
/* Grid do card de obra */ .illusia-card__body { display: grid; grid-template-columns: var(--cover-w) 1fr; grid-template-rows: auto 1fr auto auto; /* header middle tax footer */ } /* Cover spana TODAS as rows */ .illusia-card__cover { grid-column: 1; grid-row: 1 / -1; } /* Proporção da capa: sempre 2:3 */ .illusia-card__cover-link { min-height: calc(var(--cover-w) * 1.5); } /* Mobile: cover spana header + excerpt (rows 1–2). Chapters/tax/footer → grid-column: 1 / -1 */
✓ Variante: sem cover
Adicionar .illusia-card--no-cover: grid-template-columns: 1fr e uma barra lateral âmbar de 3px via ::after no body substitui visualmente a ausência da imagem.
✓ Variante: sticky
Adicionar .illusia-card--sticky: fundo levemente mais âmbar, borda inicial --border-amber em vez de --border-1, e radial-gradient sutil no canto superior esquerdo.

19 — Regras de Ouro

As 10 Regras Invioláveis

1 — Nunca valores literais no CSS
Todo espaço, fonte, cor e raio vem de um token var(--...). Valores literais só em tokens novos no :root.
2 — Toda medida de layout usa clamp()
Sem pixels fixos para espaçamento, tipografia ou raio de borda. O design vive entre 375px e 1440px sem breakpoints artificiais.
3 — Âmbar é o único acento livre
Teal, crimson e violet têm papéis semânticos rígidos (status, aviso, fandom). Nunca use para decoração.
4 — Playfair só para títulos e stats
Nunca para metadados ou labels. Fira Code só para metadados e labels. Nunca para corpo de texto.
5 — Grain sempre presente
body::before com noise SVG em position: fixed e z-index: 9999. Opacidade entre .35 e .55.
6 — Borda semântica no hover
Estado padrão: --border-1. Hover com intenção: --border-amber. Ativo/focado: --border-active. Nunca pule níveis.
7 — Tags sempre em uma linha com fade
flex-wrap: nowrap + overflow: hidden + mask-image gradiente. Nunca quebre tags em múltiplas linhas num card.
8 — Barra lateral nos items de lista
Todo row clicável tem ::before com scaleY(0) → scaleY(1) no hover usando --ease-expo. É a assinatura do sistema.
9 — Entradas escalonadas em listas
Cada item de uma lista renderizada tem animation-delay: N * 30ms. Faz o conteúdo aparecer de forma orgânica, não simultânea.
10 — Shimmer line no topo de cards
Todo card tem ::before com gradiente horizontal transparent → rgba(amber,.12) → transparent de 1px no topo. Define a superfície.
